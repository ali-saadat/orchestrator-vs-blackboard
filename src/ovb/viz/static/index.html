A company wants to hire Sam. Four people must say yes to one deal.

🙋 Sam (the Candidate) asks for $130k.

🧑‍💼 The Manager starts at $100k. That is a $30k gap. 😬

📋 HR has a rule: base salary can NOT go over the band. But nobody has said the band yet…

💰 Finance has a rule too: salary + bonus together have a hard cap.

Nobody can know the final deal now. They must negotiate — step by step, until all four say yes.

Three ways to talk 💬

Same people. Same deal. Three ways to talk together.

👔 The Boss Way (Orchestrator)

One boss asks each person. One by one. Again and again. At the end, the boss asks everyone one more time, just to check.

📋 The Blackboard Way (shared board)

All four share one blackboard. When a number changes, only the right people speak again.

🤝 The Mix Way (Hybrid)

Candidate, Manager and HR settle the salary together on the board. Then Finance signs once.

⭐ All three reach the same deal. But more talk costs more time and more money.

Your guess: which way needs the least talk?

🤔 Why is the deal always the same? Isn’t AI random?

Good question — AI models are random. Ask twice, get different words. So why do all three ways always land on the exact same deal?

Because each agent here has two parts:

🧮 A fixed rule decides the numbers — come down a step, go up a step, stop at HR’s band, sign the bonus. This is plain code. It gives the same answer every run, in any speaking order.

🗣️ The AI model only says the words for each move. Real calls, real tokens, real money — but the words never change the numbers.

A plain-code gate — never the model — checks “is the deal done?”. So the deal is fixed by design, and the three ways can differ in only one thing: how much talk it took — turns, tokens, money. That difference is the whole experiment. If the model also decided the numbers, you could not tell “different way of talking” from “different luck”.

🗣️ Want to see it without the rules? Open 🎛 Change the numbers and switch to Free talk — the AI decides every move (needs your own API key, see the README). The ways may land on different deals, or no deal at all. Run it twice: it can change. That is non-determinism, live.

📜 For experts: the real prompts each agent gets

Each turn = one streaming API call: the agent’s persona is the system prompt, the live state is the user prompt, and the model narrates the move the rule already made. Replay mode (“cassette”) re-serves the recorded answers byte for byte, so the demo is reproducible offline. Source: src/ovb/domain/agents.py (roles + rules) and src/ovb/core/registry.py (prompt assembly).

SYSTEM 🙋 Candidate: You are the Candidate. Negotiate the salary you deserve, conceding step by step; accept the band cap when it is final.

SYSTEM 🧑‍💼 Manager: You are the hiring Manager. Raise your offer step by step, never above HR's band; close the deal when you meet.

SYSTEM 📋 HR: You are HR. Announce the hard salary band, and grant remote days for the candidate's concession (1 day per $5k, 1-5).

SYSTEM 💰 Finance: You are Finance. Announce the total-pay cap and approve the signing bonus once the base is agreed (up to $8k, if room).
